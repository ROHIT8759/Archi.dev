## Spacing Scale

Declared values (multiples of 4 only). Source: observed in codebase + 8-point standard.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding inline |
| sm | 8px | Node header gaps, button icon spacing |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section internal padding |
| xl | 32px | Card-to-card gap in grids |
| 2xl | 48px | Major section vertical breaks |
| 3xl | 64px | Page-level top/bottom padding |

**Exceptions:**
- Touch targets for icon-only buttons: 44px minimum (confirmed: avatar button is 42×42, logo block is 44×44 on compact viewport — round up all icon-only targets to 44px)
- Node minimum width: 220px (confirmed from ApiEndpointNode.tsx)
- Left sidebar default width: 236px (confirmed from WorkspaceCanvas.tsx)
- Inspector default width: 320px (confirmed from WorkspaceCanvas.tsx)
- Navbar pill padding at scroll: 24px horizontal (confirmed from Navbar.tsx)
- Border radius system: 8px (nodes), 10px (buttons/inputs), 12px (tab buttons), 16px (logo block), 18px (profile menu), 24px (bento cards), 32px (auth card), 999px (pills/badges)

- Primary text: `var(--foreground)` = `#eef4ff` (near-white)
- Secondary text: `var(--secondary)` = `#c5d1e6` (muted white)
- Muted text: `var(--muted)` = `#8e9ab2` (placeholder, timestamps)
- Disabled/decorative: `rgba(255,255,255,0.25)` and below
- Gradient text (hero, logo): white → rgba(255,255,255,0.4) via `.text-gradient`

## Micro-Interaction Contract

All interactive elements must satisfy this contract. Source: PRD §1, confirmed patterns in StudioHeader.tsx.

| Trigger | Property | Value |
|---------|----------|-------|
| All interactive hover | transition duration | 300ms (PRD mandate); existing globals use 180ms — upgrade primary elements to 300ms |
| Button hover | transform | `scale(1.02)` |
| Button hover | box-shadow | `0 12px 28px rgba(3,10,22,0.34)` (`.magnetic-btn:hover`) |
| Button active | transform | `translateY(0.5px)` |
| Sidebar item hover | transform | `translateY(-2px)` |
| Node hover (`.ermiz-node:hover`) | transform | `translateY(-1px)` |
| Node selected | border | `var(--primary)` + `box-shadow: 0 0 0 1px var(--primary)` |
| Magnetic button | mouse-tracking | CSS `--mx`/`--my` custom props + radial-gradient `hover-trail::after` |
| Nav pill morph | framer-motion | borderRadius 0→999px, maxWidth 1280→1000px over 400ms `[0.16,1,0.3,1]` |
| Nav hide on scroll down | framer-motion | y: -100, opacity: 0 |
| Focus ring | outline | `2px solid var(--primary)`, offset 2px, `box-shadow: 0 0 0 4px rgba(121,183,255,0.14)` |

### Animation Easing Standard
Use `[0.16, 1, 0.3, 1]` (custom spring cubic-bezier) for all entrance/exit animations.
Use `ease-out` with `duration-300` for hover transitions.

### Performance Mandate
60fps minimum. Prefer CSS transforms and opacity for animations. No `top`/`left`/`width`/`height` transitions on hot paths. Use `will-change: transform` on sidebar items and magnetic buttons.

### 1. Landing Page (`/`)

**Navbar:**
- Default: transparent, full-width
- Scrolled > 20px: `.glass-panel` pill, borderRadius 999px, maxWidth 1000px, y: +16px (floated)
- Scroll down > 100px: hide (y: -100, opacity: 0)
- Scroll up: reveal
- Nav link hover: cyan underline indicator via framer-motion `layoutId="nav-underline"`

**Hero:**
- Background: `.bg-grid` + `.bg-noise` + two animated radial-gradient orbs (cyan top-right, violet bottom-left)

- Entrance animation: `y: 24, opacity: 0, filter: blur(8px)` → clear, 800ms, `[0.16,1,0.3,1]`
- Stats row: animated counter on inView (1800ms cubic easing)

**Bento Grid:**
- 4-column grid on desktop, 2-col tablet, 1-col mobile
- Each card: `.bento-card` (confirmed in globals.css)
- Mouse-tracking spotlight: `radial-gradient` following cursor via `--mx`/`--my` custom CSS props on each card
- Card hover: border brightens via spotlight, no additional scale (spotlight is the hover reveal)

**Sticky Scroll Sequence ("How It Works"):**
- 300vh scroll container
- Left text: 3 states ("Draw Nodes" → "AI Scaffolds" → "Deploy"), fade in/out mapped to scrollYProgress
- Right panel: simulated React Flow canvas with nodes/edges appearing at scroll thresholds

**Magnetic CTA Footer:**
- Full-width section, bottom-glowing radial gradient (cyan)
- "Start Building" button: framer-motion magnetic pull toward cursor within 80px radius
- On hover: scale(1.03), glow intensifies

## Accessibility Baseline

| Requirement | Implementation |
|------------|----------------|
| Focus rings | `*:focus-visible` → 2px `var(--primary)` outline, confirmed in globals.css |
| Color contrast — body text | `#eef4ff` on `#040507` = 16.8:1 (exceeds WCAG AA 4.5:1) |
| Color contrast — muted text | `#8e9ab2` on `#040507` = 5.1:1 (passes WCAG AA for large text; borderline for small — use muted only at 12px+ with semibold) |
| Cyan on black | `#00F0FF` on `#000000` = 12.6:1 (passes) |
| Touch targets | Minimum 44×44px for all interactive elements |
| Keyboard navigation | All interactive elements must be reachable via Tab; Command Palette must close on Escape |
| ARIA | Buttons without visible text require `aria-label`; dialogs require `role="dialog"` + `aria-labelledby` |
| Reduced motion | Wrap `framer-motion` animations in `useReducedMotion()` check; disable parallax and pulse animations when true |
